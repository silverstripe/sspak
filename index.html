SSPak

SSPak is a database and asset bundling tool for SilverStripe.

Usage: Linux and OSX

Method 1: Install script

Use the install script, where the final argument is the directory where the binary will be installed to.

curl -sS http://silverstripe.github.io/sspak/install | php -- /usr/local/bin

If the final argument is omitted, the script will be installed into the current directory. If you don't have permission to write to the directory, "sudo" will be used to escalate permissions.

Method 2: Direct download

If you prefer not to use the installer, you can download the script and copy it to your executable path:

chmod +x sspak.phar
sudo mv sspak.phar /usr/local/bin/sspak

You should now be able to invoke sspak and see the help message:

sspak
		SSPak: manage SilverStripe .sspak archives.
		...

Usage: Windows

Unfortunately sspak is not supported on Windows at this time. However it is possible to access and create sspak archives manually using freely available tools. "sspak" format relies on tar and gzip, so you will be able to work with it by either using msysgit bash and Cygwin, or by using GUI tools such as 7-zip.

Accessing existing archive

Change the "sspak" extension of the archive to "tgz". This should make it possible to open it with 7-zip. You will see it contains two further archives: "database.sql.gz" and "assets.tar.gz". Unpack these, and you should be able to access them using the same tool.

If you wish to use one of the terminal emulators, you can do the following:

tar xzf your-archive.sspak
gunzip database.sql.gz
tar xzf assets.tar.gz

This will result in a "database.sql" dump and "assets" directory.

Creating a new archive

If you wish to build an sspak archive for submission, you can do it by hand by creating the following structure:

  • Tar and gzip the entire assets directory (including the "assets" directory itself) into "assets.tar.gz"
  • Dump the database into "database.sql" file, then gzip it to obtain "database.sql.gz"
  • Tar and gzip these two files into a file with "sspak" extension
  • This file is ready for submission

If you are using one of the terminal emulators, you can achieve this as follows:

tar czf assets.tar.gz assets
gzip database.sql
tar czf your-archive.sspak assets.tar.gz database.sql.gz

"your-archive.sspak" is now ready for submission.
